Background

            At times you'll find that when you go to open your .ns1 file in Netstumbler it will spit back and error and not open. Well that's why you've got this program.

 

            Unlike NetStumbler the ns1 file is saved in the same version format as it was opened. You’ll need to open and save the file in Netstumbler if you want to update the version to the current.

 

The program will read and recover Netstumbler File Version 6 and newer

FAQ

 

My File is huge! Is there anyway I can reduce the size and still keep the important data?

            Yes, you can use the Clear All Apdata/GPSData  option. This will remove all the data used for the graph display in Netstumbler. This will

            significantly reduce the size of the file. You’ll still have the last captured signal strength and GPS data.

 

Why don’t the icons show up in Google maps?

            Did you unzip the icons.zip file in the NS1 Recovery installation directory to your Google Maps Res directory?

 

What's this <no ssid> SSID?

            There was no SSID for the record and I need one so I replace it with <no ssid>

What does clicking a checkbox do?

It fixes the most common type of corruption which is ApData corruption

 

If the Listview is showing all BSSIDs (you clicked on the Root Node (SSID), then the checkbox feature is enabled. Setting the checkbox for an item in the listview will do the following:

  • Clear the Apdata structure for that BSSID
  • Save the File to a .ns1 format
  • Open the File in Netstumbler
  • Restore the ApData structure for that BSSID

So, here’s how it’s used. If you have a file that you know is corrupt but you don’t know what BSSID is corrupt you use the checkboxes. Start with the last BSSID and check its checkbox. Netstumbler will try to open the file.

 

If it opens you’ve found the corrupt ApData structure. In the treeview for this BSSID right click and select remove ApData for entry, save the file. You’ve now recovered your file.

 

 

If it doesn’t open uncheck the BSSID and move up to the next one.

 

Eventually you should find the BSSID that have the corrupt structure.
